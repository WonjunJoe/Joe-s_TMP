5. 14개 스킬 상세

1. brainstorming

트리거: 기능 생성, 컴포넌트 구축, 동작 변경 등 모든 창의적 작업 전
컨텍스트 탐색 명확화 질문 (1개씩) 2~3개 접근법 제안 섹션별 설계 승인 설계 문서 저장 자체 리뷰
핵심 규칙: 설계 승인 없이 구현 절대 불가 (Hard Gate)

2. writing-plans

트리거: 스펙/요구사항이 있고 다단계 작업이 필요할 때
파일 구조 매핑 2~5분 단위 태스크 분할 계획서 작성 (파일/코드/명령/결과) 자체 리뷰 실행 방식 선택
핵심 규칙: placeholder 금지 ("TBD", "TODO", "Task N과 유사" 불허)

3. executing-plans

트리거: 작성된 구현 계획이 있고 별도 세션에서 실행할 때
계획서 로드 비판적 리뷰 태스크별 순차 실행 검증 브랜치 마무리
핵심 규칙: main/master에서 직접 구현 금지 (명시적 동의 없이)

4. subagent-driven-development

트리거: 구현 계획이 있고 태스크가 독립적이며 같은 세션에서 실행할 때
태스크 추출 구현자 디스패치 1차 리뷰: 스펙 준수 2차 리뷰: 코드 품질 최종 리뷰
핵심 규칙: 태스크당 2단계 리뷰 필수, 병렬 구현자 디스패치 금지

5. test-driven-development

트리거: 기능 구현, 버그 수정, 리팩토링, 동작 변경 시
RED: 실패 테스트 작성 GREEN: 최소 코드로 통과 REFACTOR: 정리
핵심 규칙: 실패 테스트 없이 프로덕션 코드 작성 금지. 테스트 전 코드 작성 시 삭제 후 재시작

6. systematic-debugging

트리거: 버그, 테스트 실패, 예상치 못한 동작, 성능 문제, 빌드 실패
1단계: 근본 원인 조사 2단계: 패턴 분석 3단계: 가설 & 테스트 4단계: 구현
핵심 규칙: 근본 원인 조사 없이 수정 시도 금지. 3회 실패 시 아키텍처 문제로 전환

7. requesting-code-review

트리거: 태스크 완료, 주요 기능 구현, merge 전, 복잡한 버그 수정 후
git SHA 확인 code-reviewer 디스패치 피드백 분류 및 처리
핵심 규칙: Critical 이슈는 반드시 즉시 수정. "단순하니까 생략" 금지

8. receiving-code-review

트리거: 코드 리뷰 피드백을 받았을 때
READ UNDERSTAND VERIFY EVALUATE RESPOND IMPLEMENT
핵심 규칙: 동조적 반응 금지. 기술적으로 틀리면 근거와 함께 반박

9. verification-before-completion

트리거: 작업 완료/수정/통과를 주장하기 직전
IDENTIFY RUN READ VERIFY CLAIM
핵심 규칙: 증거 없이 완료 주장 금지. "should", "probably" 사용 금지

10. using-git-worktrees

트리거: 격리가 필요한 기능 작업 시작 시, 구현 계획 실행 전
디렉토리 선택 .gitignore 검증 worktree 생성 셋업 실행 테스트 기준선 검증
핵심 규칙: .gitignore 검증 없이 worktree 생성 금지

11. finishing-a-development-branch

트리거: 구현 완료, 테스트 통과, 통합 준비 완료 시
테스트 실행 기본 브랜치 확인 4가지 옵션 제시 실행 & 정리
핵심 규칙: 테스트 실패 시 옵션 제시 불가. 폐기 시 "discard" 확인 필요

12. dispatching-parallel-agents

트리거: 2개 이상의 독립적 태스크, 공유 상태 없음, 순차 의존성 없음
독립 도메인 식별 에이전트별 태스크 생성 병렬 디스패치 결과 리뷰 & 충돌 확인
핵심 규칙: 에이전트 간 변경이 간섭 가능하면 사용 금지

13. using-superpowers

트리거: 모든 대화 시작 시 (자동)
스킬 적용 가능성 판단 Skill tool 호출 스킬 지시 따르기
핵심 규칙: 1%라도 관련 스킬이 있으면 반드시 호출. 질문 전에도 스킬 체크 필수

14. writing-skills

트리거: 새 스킬 생성, 기존 스킬 편집, 배포 전 검증
RED: 스킬 없이 테스트 GREEN: 최소 스킬 작성 REFACTOR: 우회 패턴 차단
핵심 규칙: 실패 테스트 없이 스킬 작성 금지 (TDD 원칙 동일 적용)
