## 1. 개념 개요

**AI Agent**란 주어진 목표를 달성하기 위해 스스로 계획을 세우고, 행동을 선택하며, 환경과 상호작용하는 자율적인 AI 시스템이다.

단순한 질문-응답(Q&A) 모델과의 핵심 차이는 **자율성(Autonomy)** 과 **행동 능력(Action)** 에 있다.

| 구분 | 일반 LLM | AI Agent |
|------|----------|----------|
| 입력 | 프롬프트 | 목표(Goal) |
| 출력 | 텍스트 응답 | 행동 + 결과 |
| 기억 | 없음 (컨텍스트 내) | 장기 메모리 가능 |
| 도구 사용 | 제한적 | 적극적 |
| 자율성 | 낮음 | 높음 |

### 핵심 구성 요소

```
┌─────────────────────────────────────────┐
│                AI Agent                  │
│                                          │
│  ┌──────────┐    ┌──────────────────┐   │
│  │  인식     │───>│  추론 / 계획     │   │
│  │ Perceive │    │  Reason / Plan   │   │
│  └──────────┘    └────────┬─────────┘   │
│                           │              │
│  ┌──────────┐    ┌────────▼─────────┐   │
│  │  메모리   │<───│  행동 / 실행     │   │
│  │  Memory  │    │  Act / Execute   │   │
│  └──────────┘    └──────────────────┘   │
└─────────────────────────────────────────┘
```

- **인식(Perceive)**: 텍스트, 이미지, 파일, 외부 API 등 환경으로부터 정보 수집
- **추론/계획(Reason/Plan)**: 목표 달성을 위한 단계 분해 및 전략 수립
- **행동(Act)**: 도구 호출, 코드 실행, 웹 검색 등 실제 행동 수행
- **메모리(Memory)**: 단기(컨텍스트), 장기(외부 저장소) 정보 유지

## 2. 발전 역사

### 2.1 초기 규칙 기반 에이전트 (1950s~1990s)
- **1956**: 다트머스 회의에서 AI 개념 정립
- **1966**: ELIZA — 규칙 기반 대화 시스템의 시초
- **1980s**: 전문가 시스템(Expert System) — 도메인 규칙을 하드코딩한 에이전트
- **한계**: 유연성 부족, 규칙 외 상황 대처 불가

### 2.2 강화학습 기반 에이전트 (1990s~2010s)
- **1997**: Deep Blue — 체스 세계 챔피언 격파 (탐색 알고리즘)
- **2013**: DQN (DeepMind) — Atari 게임에서 인간 수준 달성
- **2016**: AlphaGo — 강화학습 + 딥러닝 결합
- **특징**: 환경과의 상호작용으로 스스로 학습하는 에이전트

### 2.3 LLM 기반 에이전트의 등장 (2020~2022)
- **2020**: GPT-3 공개 — 언어 이해·생성 능력의 비약적 향상
- **2021**: GitHub Copilot — 코드 자동완성 에이전트
- **2022**: InstructGPT, ChatGPT — 지시 따르기(instruction-following) 능력 강화

### 2.4 Tool-using Agent 시대 (2023)
- **2023.03**: GPT-4 출시, 멀티모달 지원
- **2023.03**: **ReAct** 패턴 등장 — Reasoning + Acting 반복 루프
- **2023.04**: **AutoGPT**, **BabyAGI** — 최초의 자율 에이전트 오픈소스
- **2023.07**: Claude 2, LangChain 에이전트 프레임워크 대중화
- **2023.11**: OpenAI Functions → Tools API 표준화

### 2.5 멀티 에이전트 & 오케스트레이션 (2024~현재)
- **2024.01**: CrewAI — 역할 기반 멀티 에이전트 프레임워크
- **2024.03**: Claude 3 Opus — 복잡한 추론과 장문 컨텍스트 지원
- **2024.06**: Anthropic MCP(Model Context Protocol) 발표
- **2024.11**: MCP 오픈소스 공개, 에이전트 생태계 표준화 가속
- **2025**: Agentic AI 주류화 — 코딩, 리서치, 업무 자동화 에이전트 확산
- **현재**: 오케스트레이터-서브에이전트 계층 구조, 병렬 실행, 자기검증 루프 등 고도화

## 3. 기술적 특징

### 3.1 ReAct 패턴 (Reasoning + Acting)

에이전트의 가장 핵심적인 동작 패턴.

```
Goal → Thought → Action → Observation → Thought → Action → ... → Final Answer
```

1. **Thought**: 현재 상황 분석, 다음 행동 계획
2. **Action**: 도구 호출 (검색, 코드 실행, 파일 읽기 등)
3. **Observation**: 도구 실행 결과 수신
4. 목표 달성까지 반복

### 3.2 메모리 아키텍처

| 메모리 종류 | 설명 | 구현 방법 |
|------------|------|----------|
| 인컨텍스트 메모리 | 현재 대화 창 내 정보 | 컨텍스트 윈도우 |
| 외부 메모리 | 영구 저장, 검색 가능 | 벡터 DB, 파일 시스템 |
| 에피소딕 메모리 | 과거 경험/대화 기록 | 로그, 세션 저장소 |
| 시맨틱 메모리 | 지식, 사실 정보 | Knowledge Graph, RAG |

### 3.3 계획(Planning) 전략

- **Chain-of-Thought (CoT)**: 단계별 사고 과정 명시
- **Tree of Thought (ToT)**: 여러 경로를 탐색 후 최적 선택
- **Plan-and-Solve**: 전체 계획 수립 후 순차 실행
- **MCTS 기반**: 몬테카를로 트리 탐색으로 최적 경로 탐색

### 3.4 멀티 에이전트 시스템

```
┌─────────────────────────────────────────────┐
│           Orchestrator Agent                 │
│         (계획, 태스크 분배, 통합)              │
└──────┬──────────────┬─────────────┬──────────┘
       │              │             │
   ┌───▼───┐      ┌───▼───┐    ┌───▼───┐
   │ Agent │      │ Agent │    │ Agent │
   │  (A)  │      │  (B)  │    │  (C)  │
   │ 검색  │      │ 코딩  │    │ 검증  │
   └───────┘      └───────┘    └───────┘
```

- **병렬 실행**: 독립 태스크를 동시에 처리해 속도 향상
- **역할 분리**: 전문화된 서브에이전트로 품질 향상
- **자기검증**: 별도 에이전트가 결과물 검토

### 3.5 에이전트 루프

```python
while not goal_achieved:
    observation = perceive(environment)
    thought = reason(observation, memory, goal)
    action = plan(thought)
    result = execute(action)
    memory.update(result)
    goal_achieved = evaluate(result, goal)
```

## 4. 활용 예시

### 4.1 소프트웨어 개발
- **코드 작성 에이전트**: 요구사항 → 설계 → 구현 → 테스트 자동화
- **버그 수정 에이전트**: 오류 로그 분석 → 원인 파악 → 패치 적용 → 검증
- **코드 리뷰 에이전트**: PR 분석, 보안 취약점 탐지, 개선 제안
- **예**: Claude Code, GitHub Copilot Workspace, Devin

### 4.2 리서치 & 분석
- 웹 검색 + 문서 요약 + 보고서 자동 생성
- 논문 수집 → 핵심 내용 추출 → 비교 분석표 작성
- 경쟁사 분석, 시장 조사 자동화
- **예**: Perplexity, OpenAI Deep Research

### 4.3 업무 자동화
- 이메일 분류 → 답장 초안 작성 → 캘린더 일정 등록
- 데이터 수집 → 정제 → 시각화 → 슬랙 보고
- 고객 문의 자동 분류 및 1차 응대
- **예**: Zapier AI, Make.com AI Agents

### 4.4 개인 비서 (Personal Assistant)
- 일정 관리, 이메일 처리, 정보 검색
- 쇼핑, 예약, 티켓 구매 대행
- 학습 계획 수립 및 진도 관리
- **예**: ChatGPT Operator, Rabbit r1, Humane AI Pin

### 4.5 과학 & 의료
- 신약 후보 물질 탐색 에이전트
- 의료 영상 분석 + 진단 보조
- 기후 모델링 시뮬레이션 자동화
- **예**: AlphaFold 3, Google Med-PaLM 2

### 4.6 게임 & 시뮬레이션
- NPC AI — 동적 대화, 상황 인식 행동
- 게임 밸런싱 자동 테스트 에이전트
- 가상 환경 시뮬레이션 (자율주행 훈련)
- **예**: NVIDIA ACE, Inworld AI

## 5. 앞으로 발전 방향

### 5.1 더 강력한 자율성
- 장기 프로젝트를 수일~수주에 걸쳐 독립적으로 수행
- 인간 개입 없이 목표 달성까지 스스로 탐색·수정
- 자기개선(Self-improvement) 루프: 피드백으로 행동 패턴 최적화

### 5.2 멀티모달 & 물리 세계 통합
- 시각(카메라), 청각(마이크), 촉각(로보틱스) 입력 통합
- 물리적 로봇과 결합한 Embodied AI
- AR/VR 환경에서 공간 인지 기반 행동
- **예**: Figure AI, Boston Dynamics + AI

### 5.3 에이전트 네트워크 & 경제
- 수천 개의 에이전트가 협업하는 대규모 멀티에이전트 시스템
- 에이전트 간 자율 계약, 서비스 거래 (Agent Economy)
- 인간-에이전트 혼합 팀(Human-AI teaming) 표준화

### 5.4 표준화 & 상호운용성
- MCP와 같은 프로토콜의 범용화로 에이전트 간 통신 표준 정립
- 에이전트 마켓플레이스 — 플러그앤플레이 방식으로 능력 확장
- 에이전트 정체성(Identity), 권한(Permission), 감사(Audit) 체계 구축

### 5.5 신뢰성 & 안전성
- Hallucination 감소 — 사실 검증 서브에이전트 내재화
- 행동 전 인간 승인 요구 (Human-in-the-loop) 세분화
- 에이전트 행동 감사 로그, 설명 가능성(XAI) 강화
- AI 에이전트 규제 프레임워크 정립 (EU AI Act 등)

### 5.6 개인화 & 지속 학습
- 사용자별 장기 메모리로 점점 더 개인화된 에이전트
- 온디바이스(On-device) 에이전트 — 프라이버시 보호
- Few-shot, RLHF 기반 지속적인 행동 개선

## 6. Skill / Tool / MCP와의 차이점

이 네 가지는 흔히 혼용되지만, 역할과 추상화 수준이 다르다.

### 6.1 개념 비교 요약

```
┌─────────────────────────────────────────────────────────────┐
│                        AI Agent                              │
│         (자율적 목표 달성, 계획+행동+메모리 통합)              │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────┐  │
│  │    Skill     │  │    Tool      │  │       MCP        │  │
│  │ (행동 템플릿) │  │ (단일 기능)  │  │ (컨텍스트 연결)  │  │
│  └──────────────┘  └──────────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 상세 비교표

| 구분 | AI Agent | Skill | Tool | MCP |
|------|----------|-------|------|-----|
| **정의** | 자율적으로 목표를 달성하는 시스템 | 특정 작업의 실행 절차/템플릿 | 단일 기능을 수행하는 함수/API | 외부 컨텍스트를 AI에 연결하는 프로토콜 |
| **자율성** | 높음 (스스로 계획) | 낮음 (정해진 절차) | 없음 (호출 시만 작동) | 없음 (데이터 제공) |
| **상태 유지** | 가능 (메모리) | 없음 | 없음 | 없음 (stateless) |
| **목적** | 복잡한 목표 달성 | 반복 워크플로우 자동화 | 특정 기능 수행 | 외부 시스템 연동 |
| **구성** | Tool + Skill + Memory + Plan | 프롬프트 + 절차 정의 | 함수 시그니처 + 실행 로직 | 서버-클라이언트 인터페이스 |
| **예시** | Claude Code, AutoGPT | `/commit`, `/review-pr` | `web_search()`, `read_file()` | Filesystem MCP, GitHub MCP |

### 6.3 각각의 역할

#### Tool (도구)
- **무엇인가**: AI가 호출할 수 있는 **단일 기능 단위**
- **특징**: 입력을 받아 결과를 반환하는 함수. 상태 없음, 목적 하나.
- **예**: `search_web(query)`, `execute_code(code)`, `read_file(path)`
- **비유**: 망치, 드라이버 같은 단일 도구

```python
# Tool 예시
def search_web(query: str) -> str:
    """웹에서 정보를 검색하고 결과를 반환"""
    return search_engine.search(query)
```

#### Skill (스킬)
- **무엇인가**: 반복적으로 사용되는 **작업 절차의 템플릿**
- **특징**: 여러 도구와 단계를 조합한 재사용 가능한 워크플로우. 프롬프트 + 절차로 정의.
- **예**: `/commit` (git 상태 확인 → 메시지 작성 → 커밋), `/review-pr` (PR 분석 → 피드백 작성)
- **비유**: 레시피 (재료=Tool, 순서=절차)

```markdown
# Skill: commit
1. git status 확인
2. git diff 분석
3. 커밋 메시지 초안 작성
4. 사용자 확인 후 git commit 실행
```

#### MCP (Model Context Protocol)
- **무엇인가**: AI 모델과 외부 시스템을 연결하는 **표준 프로토콜**
- **특징**: 외부 데이터·서비스를 AI의 컨텍스트로 가져오는 연결 레이어. Anthropic이 2024년 공개한 오픈 표준.
- **예**: Filesystem MCP (파일 접근), GitHub MCP (저장소 연동), Slack MCP (메시지 읽기/쓰기)
- **비유**: USB 표준 (어떤 장치든 같은 방식으로 연결)

```json
// MCP 서버 정의 예시
{
  "name": "filesystem",
  "tools": [
    { "name": "read_file", "description": "파일 내용 읽기" },
    { "name": "write_file", "description": "파일 내용 쓰기" }
  ]
}
```

#### AI Agent (에이전트)
- **무엇인가**: Tool, Skill, MCP를 **자율적으로 선택·조합하여 목표를 달성**하는 시스템
- **특징**: 계획 수립, 도구 선택, 실행, 검증, 반복의 전체 루프를 스스로 관리
- **예**: "이 코드베이스의 버그를 찾아 수정하고 테스트까지 통과시켜" → 스스로 파일 탐색, 코드 분석, 수정, 테스트 실행
- **비유**: 요리사 (레시피=Skill, 도구=Tool, 식재료 공급=MCP를 자율 활용)

### 6.4 계층 관계

```
AI Agent
├── 목표 설정 & 계획 수립
├── Skill 선택 및 실행
│   ├── Tool A 호출 (via MCP or direct)
│   ├── Tool B 호출
│   └── Tool C 호출
├── MCP 서버 연동
│   ├── Filesystem MCP
│   ├── GitHub MCP
│   └── Database MCP
├── 결과 평가 & 메모리 업데이트
└── 목표 달성 여부 판단 → 반복 or 종료
```

### 6.5 언제 무엇을 쓰는가?
